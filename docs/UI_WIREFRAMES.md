# UI Wireframes (Low-Fi)

Документ для быстрого макетирования экранов в формате, понятном и человеку, и AI-агенту.

## Общие правила

- Язык интерфейса: только русский.
- Время в UI: пользовательская timezone.
- Основной паттерн действий: явный `Сохранить`, без autosave.
- Для долгих операций: job-based статус, отдельная страница `Jobs`.
- Для редактирования задач: `Revert to previous revision` + `Retry` при ошибках.

## Navigation Model

### Глобальная навигация (sidebar)

1. `Dashboard`
2. `Задачи` (`Assignments List`)
3. `Кампании` (`Batch Runs`)
4. `Очередь оператора`
5. `Проекты`
6. `Люди`
7. `Шаблоны и политики`
8. `Jobs`
9. `Настройки`

### Базовые маршруты

- `/dashboard`
- `/assignments`
- `/assignments/:assignmentId`
- `/campaigns`
- `/campaigns/:campaignId/preview`
- `/operator-queue`
- `/projects`
- `/projects/:projectId`
- `/people`
- `/templates`
- `/jobs`
- `/settings`

### Ключевые переходы между экранами

- `Dashboard` -> `Assignment Details` (клик по строке/карточке проблемы).
- `Assignments List` -> `Assignment Details` (кнопка `Открыть`).
- `Projects` -> `Assignments List` с предустановленным фильтром `project_id`.
- `Batch Runs` -> `Campaign Preview & Edit`.
- `Campaign Preview & Edit` -> `Jobs` (по ссылке из статуса отправки/job).
- `Operator Queue` -> `Assignment Details` (открыть связанный объект).
- `Jobs` -> связанный объект (`Assignment Details` или `Campaign Preview & Edit`).

### Внутриэкранная навигация

- В `Assignment Details` действия календаря (`Find slot` / `Schedule meeting`) открываются в modal/side panel без ухода со страницы.
- В `Assignment Details` секции `TouchpointTimeline`, `Evidence`, `Audit`, `Technical` работают как раскрываемые блоки.
- В `Campaign Preview & Edit` переход между сообщениями выполняется в левой таблице без смены route.

### Правила сохранения контекста

- Все фильтры и сортировки списков хранятся в query params URL.
- Кнопка браузера `Назад` возвращает в список с теми же фильтрами, страницей и сортировкой.
- Для `Assignments List` применяются default filters только при первом входе или явном `Сбросить`.

### Unsaved changes и конфликты

- При попытке уйти со страницы с несохраненными изменениями показывается confirm dialog:
  - `Сохранить и выйти`
  - `Выйти без сохранения`
  - `Отмена`
- При `CONFLICT_EDIT` сохранение блокируется, дефолтное действие — `Reload latest`.

### Job-based навигация

- Для job-based операций не делать принудительный redirect.
- После запуска job показывать toast/panel со статусом и ссылкой `Открыть в Jobs`.
- При завершении job пользователь может:
  - остаться на текущем экране и обновить данные;
  - перейти к связанному объекту через `Jobs`.

### Breadcrumbs

- `Задачи / {Название карточки}`
- `Кампании / {Название кампании} / Preview & Edit`
- `Проекты / {Название проекта}`
- `Jobs / {Job ID}`

## Экран 1: Dashboard

### Цель

Быстро дать оператору фокус на проблемных задачах и вход в очереди действий.

### Wireframe

```text
+----------------------------------------------------------------------------------+
| [Лого] Контроль задач                                         [Пользователь]    |
+----------------------------------------------------------------------------------+
| ФИЛЬТРЫ: [Проект v] [Объект v] [Период v] [Обновить]                            |
+----------------------------------------------------------------------------------+
| KPI: [Всего активных] [Просрочено] [Без ответа] [На ручной разбор]              |
+----------------------------------------------------------------------------------+
| БЛОКИ ОЧЕРЕДЕЙ                                                                    |
| [overdue] [no_response] [verification_manual] [meeting_manual] [call]           |
|                                                                              [>] |
+----------------------------------------------------------------------------------+
| КРИТИЧНО СЕГОДНЯ                                                                 |
| # | Название | Проект | Объект | Дедлайн | Статус | Приоритет | CTA [Открыть]   |
+----------------------------------------------------------------------------------+
```

### CTA

- `Открыть assignment`
- `Перейти в очередь`
- `Обновить данные`

### Состояния

- `loading`: скелетоны KPI + таблицы
- `empty`: "Критичных задач нет" + CTA `Открыть список задач`
- `error`: баннер с `error_code` + `Повторить`

## Экран 2: Assignments List

### Цель

Рабочий список задач с фильтрами по проекту/объекту и быстрым переходом в карточку.

### Wireframe

```text
+----------------------------------------------------------------------------------+
| Задачи                                                                            |
+----------------------------------------------------------------------------------+
| ПРЕДУСТАНОВКИ: [active_only] [current_project] [due_in_7_days]                   |
| ФИЛЬТРЫ: [Проект v] [Объект v] [Статус v] [Приоритет v] [Сбросить] [Сохранить]  |
+----------------------------------------------------------------------------------+
| # | Название | Проект | Объект | Ответственный | Дедлайн | Статус | Esc | ...   |
| 1 | ...      | ...    | ...    | ...           | ...     | ...    | ... | [Откр]|
| 2 | ...      | ...    | ...    | ...           | ...     | ...    | ... | [Откр]|
+----------------------------------------------------------------------------------+
| [<] 1 2 3 ... [>]                                                   Всего: N      |
+----------------------------------------------------------------------------------+
| [Баннер: Объект отсутствует в новом Excel, задача остается активной]             |
+----------------------------------------------------------------------------------+
```

### CTA

- `Открыть`
- `Сбросить фильтры`

### Состояния

- `empty`: "Задач не найдено по фильтрам" + `Сбросить фильтры`
- `partial`: часть строк с предупреждениями (баннер/иконки в строках)

## Экран 3: Assignment Details

### Цель

Управление жизненным циклом одной задачи и быстрые действия по next-step.

### Wireframe

```text
+----------------------------------------------------------------------------------+
| Название карточки                                                  [Сохранить]   |
| Дата: ... | Проект: ... | Объект: ... | Дедлайн: ... | Ответственный: ...       |
+----------------------------------------------------------------------------------+
| СТАТУС: [StatusBadge]  PRIORITY: [PriorityBadge]  ESC: [EscalationLevelChip]    |
+----------------------------------------------------------------------------------+
| ACTION PANEL                                                                       |
| [Send reminder] [Run verification now] [Schedule meeting] [Create call-task]      |
| (disabled actions with reason tooltip)                                             |
+----------------------------------------------------------------------------------+
| ПРОГРЕСС                                                                           |
| % completion [  ]   next_commitment_date [  ]   progress_note [...............]   |
+----------------------------------------------------------------------------------+
| ▸ Touchpoint Timeline (collapse/expand)                                            |
| ▸ Evidence (collapse/expand)                                                       |
| ▸ Audit revisions (collapse/expand) [Revert to previous revision]                 |
| ▸ Technical (collapse/expand)                                                      |
+----------------------------------------------------------------------------------+
| [Conflict banner: объект изменен в другой вкладке] [Reload latest]                |
+----------------------------------------------------------------------------------+
```

### CTA

- Primary: `Сохранить`
- Secondary: `Run verification now`, `Schedule meeting`
- Recovery: `Revert to previous revision`, `Retry`

### Состояния

- `saving`
- `save_failed` + `Retry`
- `conflict_edit` + блок сохранения + `Reload latest`
- `locked_by_user` (кто редактирует)

## Экран 4: Campaign Preview & Edit

### Цель

Контроль персонализированной рассылки перед отправкой и управление ошибками.

### Wireframe

```text
+----------------------------------------------------------------------------------+
| Кампания: ...                                                   [Approve&Send]   |
| Счетчики: ready: N | sent: N | blocked: N | review_required: N                  |
+----------------------------------------------------------------------------------+
| ЛЕВАЯ КОЛОНКА: Draft Messages                                                      |
| # | Получатель | Проект | Объект | Статус | [Открыть]                             |
+--------------------------------------+-------------------------------------------+
| ПРЕВЬЮ СООБЩЕНИЯ                      | РЕДАКТОР                                  |
| Subject: ...                          | Subject [.........................]        |
| To/CC: ...                            | To [....]  CC [....]                      |
| Body: ...                             | Body [................................]    |
| Attachments: ...                      | Attachments [ + Добавить ]                |
|                                       | [Save] [Retry failed] [Экспорт ошибок]    |
+--------------------------------------+-------------------------------------------+
| [sent payload immutable] [review-required rows blocked from send]                 |
+----------------------------------------------------------------------------------+
```

### CTA

- `Save` (явный)
- `Approve and Send` (double confirmation)
- `Retry failed`
- `Экспорт ошибок`

### Состояния

- `draft_generating`
- `ready_for_review`
- `send_in_progress`
- `partial_failed`
- `completed`

## Экран 6: Jobs

### Цель

Наблюдение за фоновыми операциями и единая точка для retry/диагностики.

### Wireframe

```text
+----------------------------------------------------------------------------------+
| Jobs                                                                              |
+----------------------------------------------------------------------------------+
| ФИЛЬТРЫ: [Тип job v] [Статус v] [Проект v] [Объект v] [Обновить]                 |
+----------------------------------------------------------------------------------+
| # | Job ID | Тип | Объект | Статус | Start | Duration | [Открыть]                |
| 1 | ...    | ... | ...    | running| ...   | 00:02:15 | [Детали]                |
+----------------------------------------------------------------------------------+
| ДЕТАЛИ JOB                                                                         |
| input payload | result | error(code/message/correlation_id)                       |
| [Retry] [Открыть связанный объект]                                                 |
+----------------------------------------------------------------------------------+
| * "Операция дольше обычного" показывается после 120 сек                           |
| * История jobs хранится 24 часа                                                    |
+----------------------------------------------------------------------------------+
```
